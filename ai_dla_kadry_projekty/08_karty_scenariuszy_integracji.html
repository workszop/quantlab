Scenariusz 1: AI + ERP
Integracja modeli AI z systemem planowania zasobów przedsiębiorstwa
Typowy czas
4–9 mies.
ERP
SAP/Oracle
⟷
Model AI
⟷
API / ETL
Middleware
→
ERP UI
Wyniki/rekomendacje
Opis przepływu danych ERP eksportuje dane transakcyjne (zlecenia, stany magazynowe, faktury) przez API lub ETL do środowiska AI. Model przetwarza dane, generuje predykcje lub rekomendacje, które są zwracane do ERP i prezentowane użytkownikom w natywnym interfejsie.
Wymagania techniczne
API ERP (REST/SOAP) Middleware / ESB Data warehouse Uprawnienia danych Środowisko ML (cloud/on-prem)
Przykładowe narzędzia / vendorzy
  • SAP BTP + SAP AI Core
  • Oracle AI Services
  • Azure ML + Azure Integration Services
  • MuleSoft / Dell Boomi
Typowe wyzwania
  • Dostęp do danych ERP często wymaga osobnych umów licencyjnych
  • Schematy danych ERP są złożone – wysoka bariera wejścia dla zespołu AI
  • Zmiany w procesach ERP mogą zepsuć integrację bez ostrzeżenia
  • Opór użytkowników biznesowych wobec "czarnej skrzynki" wydającej rekomendacje

Scoring/sugestie
Opis przepływu danych CRM dostarcza dane o interakcjach z klientami, historii transakcji i profilu. AI generuje scoring potencjalnych klientów, predykcje churnu, sugestie next-best-action oraz automatyzuje drafty e-maili. Wyniki integrują się jako pola i panele w CRM.
Wymagania techniczne
CRM API / webhooks Model scoring (ML) LLM dla NLP Historia kontaktów 12+ mies. Single Sign-On
Przykładowe narzędzia / vendorzy
  • Salesforce Einstein + Agentforce
  • HubSpot AI + OpenAI API
  • Microsoft Dynamics 365 Copilot
  • Pipedrive + Zapier + GPT
Typowe wyzwania
  • Jakość danych CRM często niska – zduplikowane kontakty, brakujące pola
  • Handlowcy traktują scoring AI jako zagrożenie dla autonomii decyzji
  • Koszt licencji na natywne AI CRM może być wysoki przy dużych zespołach
  • RODO: dane klientów w chmurze AI wymagają analizy prawnej

Scenariusz 3: AI + Narzędzia Workflow
Automatyzacja procesów przez integrację AI z platformami no-code/low-code
Typowy czas
1–4 mies.
Trigger
e-mail/formularz
→
Workflow
Zapier/Make/Power
→
AI (LLM API)
GPT/Claude
→
Akcja
CRM/Slack/Drive
Opis przepływu danych Zdarzenie (nowy e-mail, formularz, plik) wyzwala workflow w platformie automatyzacji. Workflow wysyła dane do API modelu językowego (np. OpenAI, Anthropic). AI przetwarza i zwraca wynik (klasyfikacja, podsumowanie, draft odpowiedzi), który jest zapisywany w docelowym systemie.
Wymagania techniczne
API klucz LLM Konto Zapier/Make/Power Automate Connektory do systemów Podstawy prompt engineering
Przykładowe narzędzia / vendorzy
  • Zapier + OpenAI
  • Make (Integromat) + Claude API
  • Microsoft Power Automate + Copilot Studio
  • n8n (self-hosted, open source)
Typowe wyzwania
  • Koszty API LLM mogą rosnąć nieliniowo przy dużym wolumenie
  • Brak kontroli wersji modelu – zmiana modelu przez providera może zmienić zachowanie
  • Trudno testować przepływy z AI – error handling wymaga starannego projektowania
  • Dane wysyłane do LLM przechodzą przez serwery zewnętrzne – kwestia poufności

Scenariusz 4: AI jako samodzielne narzędzie
Wdrożenie dedykowanej aplikacji AI bez głębokiej integracji z istniejącymi systemami
Typowy czas
2–5 mies.
Użytkownik
przeglądarka
→
Aplikacja AI
Frontend + Backend
⟷
Modele AI
LLM + RAG
⟷
Baza wiedzy
dokumenty / wektory
Opis przepływu danych Dedykowana aplikacja webowa lub wewnętrzna platforma AI (np. chatbot firmowy, asystent decyzyjny). Użytkownicy interagują przez interfejs, który komunikuje się z LLM i własną bazą wiedzy (RAG). Brak bezpośredniej integracji z ERP/CRM – wymiana danych ręczna lub przez pliki.
Wymagania techniczne
Hosting (cloud/on-prem) LLM API lub model lokalny Vector DB (Pinecone/Qdrant) Authorizacja użytkowników Dokumenty / baza wiedzy
Przykładowe narzędzia / vendorzy
  • Azure OpenAI + Azure AI Search
  • Anthropic Claude + własny backend
  • Ollama (lokalne modele) + LangChain
  • Copilot Studio (MS) / Botpress
Typowe wyzwania
  • Zarządzanie bazą wiedzy – dokumenty muszą być aktualne i dobrze zorganizowane
  • Halucynacje LLM – konieczne mechanizmy weryfikacji odpowiedzi
  • Adoption – pracownicy muszą zmienić nawyk i zacząć korzystać z nowego narzędzia
  • Skalowalność: koszt i latencja rosną z liczbą użytkowników i złożonością zapytań
